## 2. 設計書

**システムアーキテクチャ概要:**
本アプリケーションは、Next.jsを用いたフロントエンド主導のアーキテクチャを採用します。ユーザーインターフェースはReactコンポーネントで構成され、電子書籍閲覧ビューアや要約結果表示などを提供します。サーバサイドでは、Next.jsのAPIルートを利用して要約リクエストを代理送信する簡易バックエンドとして動作します（必要に応じて）。MVP段階ではサーバ側に状態を持たず、アップロードされたEPUBデータはブラウザ上（メモリ上）で解析・表示することを基本とします。要約機能は章単位の要約に加えて、ハイライト範囲の要約もサポートし、読書メモ機能により要約内容や理解をMarkdown形式で記録できます。

### コンポーネント設計

#### コアコンポーネント

* **ResizablePane コンポーネント:** リサイズ可能なペインを提供する汎用コンポーネント。
  ```typescript
  interface ResizablePaneProps {
    minWidth?: number;
    maxWidth?: number;
    defaultWidth: number;
    position: 'left' | 'right';
    children: React.ReactNode;
    onResize?: (width: number) => void;
  }
  ```
  - ドラッグによるサイズ調整
  - 最小/最大サイズの制限
  - ダブルクリックでのデフォルトサイズ復帰
  - レスポンシブ対応（モバイル時は非表示/タブ切り替え）

* **LLMSettingsDialog コンポーネント:** LLM設定を行うモーダルダイアログ。
  ```typescript
  interface LLMSettingsDialogProps {
    isOpen: boolean;
    onClose: () => void;
    config: LLMConfig;
    onConfigUpdate: (newConfig: LLMConfig) => void;
  }
  ```
  - APIエンドポイントとキーの設定
  - モデルパラメータの調整
  - バリデーションと即時フィードバック
  - 設定の永続化

#### ビューアコンポーネント

* **ReaderLayout コンポーネント:** 3ペイン構造のメインレイアウト。
  ```typescript
  interface ReaderLayoutProps {
    metadata: BookMetadata;
    chapters: Chapter[];
    currentChapter: number;
    content: Record<string, string>;
    onChapterSelect: (href: string) => void;
  }
  ```
  - ResizablePaneを使用した3ペイン構造
  - 目次ペイン（左）: TableOfContents
  - 本文ペイン（中央）: ContentView
  - 要約/メモペイン（右）: SummaryAndNoteView
  - レスポンシブ対応（モバイル時はタブ切り替え）

* **TextHighlighter コンポーネント:** テキスト選択とハイライト機能。
  ```typescript
  interface Highlight {
    id: string;
    startOffset: number;
    endOffset: number;
    selectedText: string;
    timestamp: number;
  }

  interface TextHighlighterProps {
    content: string;
    onHighlight: (highlight: Highlight) => void;
    activeHighlights: Highlight[];
    onHighlightClick: (id: string) => void;
  }
  ```
  - テキスト選択範囲の取得と管理
  - ハイライトの視覚的表示
  - 選択範囲の要約トリガー
  - ハイライト情報の永続化

#### メモ機能コンポーネント

* **NoteEditor コンポーネント:** Markdownエディタ。
  ```typescript
  interface Note {
    id: string;
    title: string;
    content: string;
    lastModified: number;
    chapterId?: string;
    tags?: string[];
  }

  interface NoteEditorProps {
    note: Note;
    onChange: (content: string) => void;
    onSave: () => void;
    autoSave?: boolean;
  }
  ```
  - Markdownツールバー
  - シンタックスハイライト
  - 自動保存機能
  - 編集履歴管理

* **NotePreview コンポーネント:** Markdownプレビュー。
  ```typescript
  interface NotePreviewProps {
    content: string;
    onLinkClick?: (href: string) => void;
    className?: string;
  }
  ```
  - Markdownのリアルタイムレンダリング
  - 目次ジャンプ機能
  - カスタムスタイリング

* **NoteTemplateGenerator:** メモテンプレート生成。
  ```typescript
  interface TemplateGeneratorProps {
    chapters: Chapter[];
    summaries?: Record<string, string>;
    onGenerate: (template: string) => void;
  }
  ```
  - 目次構造からのテンプレート生成
  - 要約内容の自動埋め込み
  - カスタマイズ可能なテンプレート構造

### データフローと内部処理

1. **レイアウト管理:**
   - ReaderLayoutが3つのResizablePaneを管理
   - ペインサイズの状態管理とローカルストレージでの保存
   - モバイル表示時のタブ切り替え制御
   ```typescript
   interface LayoutState {
     leftPaneWidth: number;
     rightPaneWidth: number;
     activeTab: 'content' | 'summary' | 'note';
     isMobile: boolean;
   }
   ```

2. **テキスト選択とハイライト:**
   - window.getSelection() APIによる選択範囲の取得
   - 選択範囲のオフセット計算と正規化
   - ハイライト情報の永続化
   ```typescript
   interface SelectionRange {
     startContainer: Node;
     startOffset: number;
     endContainer: Node;
     endOffset: number;
     commonAncestor: Node;
   }
   ```

3. **要約処理の流れ:**
   - 章単位の要約：
     1. 章の範囲指定（単一または複数章）
     2. 指定範囲のテキスト抽出
     3. 要約APIへのリクエスト
     4. 結果の表示と保存
   - ハイライト範囲の要約：
     1. テキスト選択とハイライト作成
     2. 選択範囲の前後コンテキスト取得
     3. 要約APIへのリクエスト（コンテキスト付き）
     4. 結果の表示とハイライトへの関連付け

4. **メモ管理の流れ:**
   - テンプレート生成：
     1. 目次構造の解析
     2. 各章見出しの生成
     3. 要約内容の配置
     4. テンプレートの適用
   - メモの保存：
     1. 自動保存（定期的/変更時）
     2. ローカルストレージへの永続化
     3. エクスポート（Markdown/PDF）
